Weekly Nerds

Minor Web Design & Development

Communication and Multimedia Design, Amsterdam

Vivanne Hoogendam

Leerdoelen

  1. Doel 1 – Werken met een duidelijke mappenstructuur in de backend
    Wat wil ik leren? Ik wil werken met een duidelijkere mappenstructuur en componenten in mijn backendproject. Denk aan het juist opsplitsen van routes, controllers en services, en het leren importeren van pagina’s of functionaliteiten op een overzichtelijke manier. Dit is nu nog onwennig voor mij.
    Waarom? Zodat ik efficiënter kan werken, fouten sneller kan opsporen en mijn code begrijpelijk is voor anderen.
    Wanneer ben ik tevreden? Als ik een backend heb opgezet waarbij mijn mappenstructuur logisch is opgebouwd, ik alles zelfstandig kan importeren, en een ander mijn structuur kan begrijpen zonder uitleg.

  2. Doel 2 – Meer grip krijgen op database-opzet en datastructuur
    Wat wil ik leren? Ik wil meer grip krijgen op het zelfstandig opzetten en structureren van een database. Ik wil snappen hoe je goede keuzes maakt in wat je opslaat en hoe je het opbouwt.
    Waarom? Omdat ik daar nog weinig ervaring mee heb en in toekomstige projecten efficiënter en doelgerichter met data wil kunnen werken.
    Wanneer ben ik tevreden? Als ik tijdens dit project bewuste keuzes heb gemaakt in mijn datastructuur (zoals welke velden en relaties ik gebruik), en als ik na afloop in mijn eigen woorden kan uitleggen waarom ik het zo heb aangepakt en wat ik een volgende keer anders zou doen.

  3. Doel 3 – Focussen op micro-interacties en afwerking
    Wat wil ik leren? Ik wil mij richten op het verfijnen van kleine interactieve momenten op de website, zoals hover-animaties, micro-transities of het zichtbaar maken van foutmeldingen op een prettige manier.
    Waarom? Omdat ik deze stap vaak oversla, terwijl het een project echt af kan maken en de gebruikservaring kan verbeteren.
    Wanneer ben ik tevreden? Als ik bij minimaal 3 onderdelen op de site bewust aandacht heb besteed aan visuele details, ik zelf merk dat het ‘af’ voelt, en ik positieve feedback krijg over de vloeiendheid van de site.

  4. Doel 4 – Rol als backender versterken én frontend verbreden
    Wat wil ik leren? Tijdens de meesterproef wil ik de rol van backender aannemen, omdat ik hier energie van krijg en er verder in wil groeien. Daarnaast wil ik ook meewerken aan een deel van de frontend.
    Waarom? Om mijn backendskills te versterken én mezelf breder inzetbaar te maken in projecten.
    Wanneer ben ik tevreden? Als ik een werkende backend heb gemaakt die goed samenwerkt met de frontend én ik een klein stuk frontend heb bijgedragen, zoals een interactief formulier of visuele component.
